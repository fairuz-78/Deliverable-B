# Deliverable B: Komunikasi Tim & Resolusi Konflik

## Team Manifesto
Tim kami berkomitmen untuk bekerja secara profesional dalam menyelesaikan proyek Aplikasi Mobile Kantin Pintar Kampus.

### Komunikasi Terbuka
Setiap anggota tim wajib menyampaikan kendala dan progress secara jelas kepada tim.

### Dokumentasi Wajib
Setiap perubahan API harus dicatat untuk menghindari miskomunikasi.

### Tidak Saling Menyalahkan
Tim fokus pada solusi, bukan menyalahkan individu.

### Diskusi Sebelum Perubahan
Semua perubahan sistem harus melalui diskusi tim.

### Daily Standup
Meeting 15 menit setiap hari untuk update progress.

---

## SOP Code Review

### Tidak Diperbolehkan:
- Menggunakan kata kasar  
- Kritik tanpa solusi  
- Menyalahkan individu  

### Dianjurkan:
- Bahasa sopan  
- Memberikan solusi  
- Fokus pada kode  

### Contoh:
❌ Kode ini jelek  
✅ Kode ini bisa diperbaiki agar lebih rapi  

---

## Protokol Komunikasi

- Jam kerja: 08.00 – 21.00 WIB  
- Tidak menghubungi di luar jam kerja (kecuali darurat)  
- Diskusi teknis via ClickUp / GitHub  
- Update progress harian wajib  

---

## Tujuan
Dokumen ini dibuat untuk memastikan komunikasi tim berjalan efektif dan mencegah konflik seperti yang terjadi pada tim sebelumnya.
